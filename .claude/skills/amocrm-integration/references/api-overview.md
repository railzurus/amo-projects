# Общие правила amoCRM API v4

## Базовый URL
```
https://{subdomain}.amocrm.ru/api/v4/
```

## Заголовки
- `Authorization: Bearer {access_token}` — обязательный для всех запросов
- `Content-Type: application/json` — для POST/PATCH запросов
- Успешный ответ: `Content-Type: application/hal+json`
- Ошибка: `Content-Type: application/problem+json`

## Лимиты
- **7 запросов в секунду** на аккаунт
- При превышении — HTTP 429
- При многократном превышении — HTTP 403 (блокировка аккаунта)
- Максимум 250 сущностей за один GET-запрос (параметр `limit`)
- Максимум 250 сущностей за один POST/PATCH-запрос (пакетное создание/обновление)
- Максимум 100 вебхуков в аккаунте

## Пагинация
Все списочные методы поддерживают `page` и `limit`:
```
GET /api/v4/leads?page=1&limit=250
```
Наличие следующей страницы определяется по `_links.next` в ответе. Если ключ отсутствует — данных больше нет.

## Коды ответов

### Успешные
| Код | Описание |
|-----|----------|
| 200 | Запрос выполнен успешно |
| 201 | Сущность создана |
| 204 | Контент отсутствует (нет данных / удалено успешно) |

### Ошибки авторизации
| Код | amoCRM код | Описание |
|-----|-----------|----------|
| 401 | 110 | Неправильный логин или пароль |
| 401 | 111 | Слишком много попыток, нужна капча |
| 401 | 112 | Пользователь выключен или не в аккаунте |
| 403 | 113 | IP не в белом списке |
| 401 | 101 | Несуществующий аккаунт (субдомен) |

### Ошибки данных
| Код | Описание |
|-----|----------|
| 400 | Неверная структура данных или неверные ID полей |
| 422 | Данные не могут быть обработаны |
| 405 | HTTP-метод не поддерживается |

### Ошибки оплаты и лимитов
| Код | Описание |
|-----|----------|
| 402 | Подписка закончилась |
| 403 | Аккаунт заблокирован за превышение лимитов |
| 429 | Превышено количество запросов в секунду |

### Ошибки валидации
При HTTP 400 возвращается структура:
```json
{
    "validation-errors": [
        {
            "request_id": "0",
            "errors": [
                {
                    "code": "NotSupportedChoice",
                    "path": "custom_fields_values.0.field_id",
                    "detail": "The value you selected is not a valid choice."
                }
            ]
        }
    ],
    "title": "Bad Request",
    "status": 400
}
```

## Параметр `with`
Многие GET-методы поддерживают параметр `with` для подгрузки связанных данных. Передаётся как строка через запятую:
```
GET /api/v4/leads?with=contacts,catalog_elements,loss_reason
```

## Фильтрация
Передаётся через GET-параметр `filter`:
```
GET /api/v4/leads?filter[statuses][0][pipeline_id]=31&filter[statuses][0][status_id]=142
GET /api/v4/contacts?filter[responsible_user_id]=504141
GET /api/v4/tasks?filter[is_completed]=0&filter[entity_type]=leads
```

## Сортировка
Передаётся через GET-параметр `order`:
```
GET /api/v4/leads?order[updated_at]=desc
GET /api/v4/tasks?order[complete_till]=asc
```

## Информация об аккаунте
```
GET /api/v4/account
```
Параметры with: `amojo_id`, `amojo_rights`, `users_groups`, `task_types`, `version`, `entity_names`, `datetime_settings`

Возвращает: id, name, subdomain, currency, timezone и другие настройки.

## Пользователи
```
GET /api/v4/users — список пользователей
GET /api/v4/users/{id} — конкретный пользователь
GET /api/v4/roles — список ролей
```

## Рекомендации по работе с API
- Всегда обрабатывай HTTP 429 с retry через 1+ секунду
- Используй пакетные операции (до 250 сущностей) вместо одиночных запросов
- Кэшируй ID полей, воронок, статусов — они не меняются часто
- При записи токенов в БД используй тип TEXT (длина токена может варьироваться)
- Для массовой выгрузки используй фильтр по `updated_at` для инкрементальной синхронизации

## Типы интеграций amoCRM

### 1. Приватная интеграция
Для одного аккаунта. Самый простой вариант: создаёшь интеграцию в amoМаркет → копируешь ключи → обмениваешься данными. Не проходит модерацию, не публикуется в маркетплейсе. Может содержать виджет (JS-код), но только для одного аккаунта.

### 2. Публичная интеграция (маркетплейс)
Для всех пользователей amoCRM. Проходит модерацию (проверка, тестирование, аудит JS-кода). Разработка и тестирование — в техническом аккаунте. После модерации доступна на странице интеграций у всех пользователей.

### 3. Интеграция с виджетом
Любая интеграция (приватная или публичная) может содержать архив виджета с JS-файлами, которые подгружаются в интерфейсе amoCRM. Виджет позволяет: отображать UI в карточках/списках/воронке, взаимодействовать с DOM amoCRM, обмениваться данными с внешними сервисами из браузера, получать данные из JS-контекста amoCRM.

### 4. Интеграция только через backend API
Без JS-виджета. Взаимодействие только через REST API: синхронизация данных, обработка вебхуков, автоматизация.
